## 1. 引言
### 1.1 文档目的
本需求说明书旨在明确“AI 学伴 - 轻量化团队 AI 辅助学习平台”的全量需求，包括功能、接口、性能、用户体验等维度，作为项目开发、测试、验收、维护的核心依据，确保开发成果符合高校师生及管理员的实际使用需求，同时满足项目开发流程规范。

### 1.2 项目背景
随着 AI+ 教育市场规模持续增长（2024 年超 3000 亿元，年增 25%），ChatGPT、文心一言等大语言模型降低了 AI 工具使用门槛，但现有产品存在“大型平台适配 K12 场景、专业工具功能单一”的问题。高校师生面临“学生答疑不及时、笔记整理效率低；教师出题/教案编写耗时、学生数据分散”等痛点。本项目聚焦高校场景，开发轻量化、多角色协同的 AI 辅助学习平台，解决上述痛点。

### 1.3 文档范围
覆盖平台的学生端、教师端、管理员端全功能模块，包含功能需求、接口需求、性能指标、用户需求、非功能需求、数据需求等，不包含项目管理（如进度、预算）相关内容。

### 1.4 术语与缩写
| 术语/缩写 | 说明 |
|-----------|------|
| AI API | 大语言模型开放接口（OpenAI API/百度文心一言/阿里通义千问） |
| FastAPI | 后端开发框架，用于构建高性能 API |
| Vue 3 | 前端渐进式框架，搭配 TypeScript 实现类型安全 |
| JWT | JSON Web Token，用于用户身份认证 |
| RESTful | 前后端交互的 API 设计规范 |
| 响应式布局 | 前端页面适配不同设备（PC/平板）的布局方式 |
| QPS | 每秒查询率，衡量接口并发处理能力 |

## 2. 总体需求
### 2.1 目标用户
- 核心用户1：高校学生（本科/研究生），需完成作业、整理笔记、分析错题、获取答疑；
- 核心用户2：高校教师，需出题、管理教案/试题、跟踪学生学习情况；
- 运维用户：平台管理员，需管理用户、配置系统、监控数据。

### 2.2 核心目标
1. 开发支持学生/教师/管理员三角色的 Web 端 AI 辅助学习平台；
2. 核心功能覆盖 AI 答疑、笔记总结、错题分析、作业批改、AI 出题、教案/试题管理；
3. 保证系统稳定、界面友好、操作简单，适配高校教学场景；
4. 集成多厂商 AI API，降低使用成本，支持轻量化部署。

### 2.3 运行环境
| 类型 | 要求 |
|------|------|
| 前端运行环境 | 主流浏览器（Chrome ≥ 90、Firefox ≥ 88、Edge ≥ 90）；响应式适配 1366×768 及以上分辨率 |
| 后端运行环境 | 服务器操作系统（Linux CentOS 7+/Ubuntu 20.04+）；Node.js ≥ 18.0.0；FastAPI 最新稳定版 |
| 数据库环境 | MySQL ≥ 5.0.0；支持远程连接、索引优化 |
| 网络环境 | 公网/校园网，带宽 ≥ 2Mbps（保证 AI API 调用稳定性） |

## 4. 接口需求
### 4.1 内部接口需求
#### 4.1.1 前后端交互接口（RESTful 规范）
| 接口类型 | 规范要求 | 示例（用户登录） |
|----------|----------|------------------|
| 请求规范 | 1. 请求方法：GET（查询）/POST（创建）/PUT（更新）/DELETE（删除）；2. 请求头：携带 Token（Authorization: Bearer {JWT}）；3. 请求参数：JSON 格式，必填项非空校验 | POST /api/v1/user/login<br>请求体：{"account":"13800138000","password":"EncryptedPwd","verifyCode":"123456"} |
| 响应规范 | 1. 响应状态码：200（成功）/400（参数错误）/401（未授权）/403（禁止访问）/500（服务器错误）；2. 响应体：统一格式 {"code":0,"msg":"成功","data":{}}；3. 错误码：自定义（如 1001=账号不存在，1002=密码错误） | {<br>"code":0,<br>"msg":"登录成功",<br>"data":{"token":"xxx","userInfo":{"id":1,"role":"student","name":"张三"}}<br>} |

#### 4.1.2 后端模块间接口
| 模块 | 交互接口 | 需求 |
|------|----------|------|
| 控制器层 ↔ 服务层 | AI 服务接口 | 1. 统一封装 AI 调用逻辑，提供通用调用方法；2. 支持切换不同 AI 服务商；3. 异常捕获与降级处理 |
| 服务层 ↔ 数据层 | 数据库操作接口 | 1. 统一的 CRUD 封装；2. 支持事务管理（如作业提交+状态更新）；3. 慢查询监控（执行时间>1s 记录日志） |

### 4.2 外部接口需求
#### 4.2.1 AI API 接口
| 服务商 | 接口需求 | 降级策略 |
|--------|----------|----------|
| OpenAI API | 1. 调用 Chat Completions 接口，支持 gpt-3.5-turbo 模型；2. 上下文长度 ≤ 4096 tokens；3. 超时时间 ≤ 10s | 调用失败时自动切换至百度文心一言 |
| 百度文心一言 | 1. 调用ERNIE-Bot 接口；2. 支持学科定制化提示词；3. 超时时间 ≤ 8s | 调用失败时自动切换至阿里通义千问 |
| 阿里通义千问 | 1. 调用通义千问通用接口；2. 支持长文本总结（≤2万字）；3. 超时时间 ≤ 8s | 调用失败时提示用户“当前 AI 服务繁忙，请稍后重试” |

#### 4.2.2 其他外部接口
| 接口类型 | 需求 |
|----------|------|
| 数据库接口 | 1. 支持 MySQL 增删改查，索引优化；2. 连接池配置（最大连接数 50，最小空闲数 10）；3. 慢查询日志开启 |
| 文件存储接口 | 1. 支持本地文件存储/对象存储（可选）；2. 文件上传/下载/删除接口；3. 文件MD5校验，防止重复上传 |
| 短信/邮箱接口 | 1. 调用第三方短信/邮箱服务商接口（如阿里云短信）；2. 验证码发送频率限制（1分钟/次）；3. 发送日志记录 |

### 4.3 UI 接口需求（前端交互）
| 交互类型 | 需求 |
|----------|------|
| 页面跳转 | 1. 路由跳转无白屏（加载态提示）；2. 权限不足跳转至403页面；3. 页面刷新保留当前状态（如错题本筛选条件） |
| 组件交互 | 1. 按钮点击反馈（加载态/禁用态）；2. 表单提交前校验，错误提示清晰（红色文字+图标）；3. 弹窗/抽屉关闭后重置表单（可选） |
| 数据可视化 | 1. 图表加载动画；2. 数据更新时平滑过渡；3. 支持图表导出（PNG/PDF） |
| 响应式适配 | 1. PC端适配不同分辨率（1366×768~2560×1440）；2. 平板端适配（横屏优先）；3. 元素布局无错位、文字无溢出 |

## 5. 性能指标需求
| 指标类型 | 详细要求 |
|----------|----------|
| 响应时间 | 1. 普通接口（登录、列表查询）：≤ 500ms；2. AI 接口（答疑、总结、出题）：≤ 3s（95% 场景）；3. 文件上传（≤10MB）：≤ 5s；4. 页面首次加载：≤ 2s（缓存后≤1s） |
| 并发处理 | 1. 支持同时在线用户数 ≥ 100；2. 接口 QPS ≥ 50；3. AI 接口并发调用 ≥ 20（无明显延迟） |
| 稳定性 | 1. 系统可用性 ≥ 99.5%（每月故障时间 ≤ 3.6小时）；2. AI 接口调用成功率 ≥ 98%（失败自动重试1次）；3. 数据库无死锁、无数据丢失 |
| 资源占用 | 1. 服务器 CPU 使用率 ≤ 70%（峰值）；2. 内存占用 ≤ 80%（峰值）；3. 磁盘使用率 ≤ 85%；4. 带宽占用 ≤ 80%（峰值） |
| 数据处理 | 1. 单批次笔记总结（5万字）：≤ 10s；2. 单批次错题分析（10道题）：≤ 8s；3. 自动组卷（50道题）：≤ 5s |

## 7. 非功能需求
### 7.1 安全性需求
| 安全维度 | 详细要求 |
|----------|----------|
| 数据安全 | 1. 用户密码加密存储（BCrypt 算法）；2. 敏感数据（手机号/邮箱）脱敏展示；3. 数据库传输加密（SSL）；4. 文件上传病毒扫描（防止恶意文件） |
| 接口安全 | 1. 接口防刷（IP+Token 频率限制，1分钟≤20次）；2. 防 SQL 注入（参数化查询）；3. 防 XSS 攻击（前端输入过滤、后端转义）；4. 防 CSRF 攻击（Token 校验） |
| 权限安全 | 1. 最小权限原则，不同角色仅拥有必要权限；2. 管理员操作日志全程记录；3. 重要操作（如数据删除）需二次确认 |

### 7.2 易用性需求

| 帮助支持 | 1. 内置帮助中心（常见问题、操作指南）；2. 功能模块提供“使用说明”弹窗；3. 支持在线反馈（问题提交、建议收集） |
