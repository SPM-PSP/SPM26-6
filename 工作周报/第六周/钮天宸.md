## 一、本周完成工作

### 1. AI 试卷生成模块开发（assessment_generator.py）
- 完成基于 RAG + LLM 的自动试卷生成模块开发（见 :contentReference[oaicite:0]{index=0}）
- 实现核心功能：
  - 基于教学内容（教案）进行试题生成
  - 支持多题型约束（选择题 / 简答题 / 编程题等）
  - 自动生成标准答案与解析
- 实现统一输出格式控制（题目区 + “参考答案与解析”分离）
- 输出：AI 试卷生成模块

### 2. RAG 检索增强能力优化
- 实现基于向量数据库（Chroma）的试卷生成检索模块
- 支持多关键词融合查询（keywords_list → query）
- 引入知识片段（RAG Snippets）增强出题质量
- 提升试题内容的准确性与知识覆盖度

### 3. Prompt 工程优化（试卷生成）
- 设计结构化 Prompt 模板：
  - 明确题型数量约束（严格控制输出）
  - 强化格式规范（题目编号、答案分区）
  - 针对编程题增加标准化输出结构（输入/输出/示例/代码解析）
- 提高模型输出稳定性与可解析性

### 4. 多轮对话与试卷修改机制实现
- 实现基于用户意图的试卷生成策略：
  - REWRITE：全量重生成试卷
  - REVISION：局部修改试卷
  - INCREMENTAL_ADD：增量添加题目
- 支持结合历史对话进行试卷优化生成
- 实现试卷“增量更新 + 完整重构”双模式

### 5. LLM 调用与生成流程优化
- 封装试卷生成调用链（Prompt → LLM → 输出解析）
- 实现统一调用接口（ChatTongyi）
- 优化生成流程稳定性与异常处理

### 6. 系统集成与测试
- 将试卷生成模块集成至后端服务层
- 与教案生成模块实现联动（教案 → 试卷）
- 完成接口测试与功能验证
- 输出：测试记录与问题修复
