## 一、项目概述

### 1.1 项目定位

**AI 学伴**是一个面向高校师生的轻量化 AI 辅助学习平台，通过集成多个大语言模型（OpenAI、百度文心一言、阿里通义千问），为学生提供智能答疑、笔记总结、错题分析、作业批改等服务，为教师提供智能出题、教案管理、学生管理等功能。

### 1.2 核心价值

| 用户角色 | 核心痛点 | 解决方案 |
|---------|---------|---------|
| **学生** | 课后答疑不及时、笔记整理效率低、错题分析缺乏针对性 | AI 即时答疑、智能笔记总结、错题智能分析 |
| **教师** | 重复性答疑工作量大、试题编写耗时、学生学习数据分散 | AI 智能出题、批量作业批改、学生学习数据可视化 |
| **管理员** | 用户管理复杂、系统监控困难 | 统一用户管理、系统配置可视化、数据统计报表 |

### 1.3 技术栈

```
前端：Vue 3 + TypeScript + Vite + Element Plus
后端：FastAPI + Python 3.10+
数据库：MySQL 8.0+
AI 服务：OpenAI API / 百度文心一言 / 阿里通义千问
部署：阿里云/腾讯云 + Docker + Nginx
```

### 1.4 项目约束

| 约束类型 | 具体内容 |
|---------|---------|
| **时间约束** | 12周开发周期（2026年3月-2026年6月） |
| **团队约束** | 6人团队（1负责人+2前端+2后端+1测试） |
| **预算约束** | 总预算 ¥970（主要成本：AI API调用） |
| **性能约束** | 普通接口≤500ms，AI接口≤3s，并发≥100用户 |
| **技术约束** | 优先使用国内AI服务，降低成本和延迟 |

### 2.2 模块划分

| 模块名称 | 功能描述 | 技术实现 |
|---------|---------|---------|
| **认证模块** | 用户注册、登录、权限控制 | JWT + RBAC |
| **学生模块** | AI答疑、笔记总结、错题分析、作业提交 | Vue 3 + FastAPI |

### 2.3 数据流规范

```
用户请求 → Nginx → FastAPI → 业务逻辑层 → 数据访问层 → MySQL
                    ↓
                AI服务层 → AI API (OpenAI/百度/阿里)
                    ↓
                缓存层 → Redis (可选)
```

## 三、功能规范

### 3.1 用户认证规范

#### 3.1.1 注册功能

**功能ID**: AUTH-001  
**功能名称**: 用户注册  
**优先级**: P0（必须）

**业务规则**:
1. 支持手机号或邮箱注册
2. 密码强度要求：≥8位，必须包含字母和数字
3. 需要验证码验证（图形验证码或短信/邮件验证码）
4. 角色选择：学生或教师（管理员仅后台创建）
5. 教师注册需管理员审核

**输入规范**:
```json
{
  "account": "13800138000",        // 手机号或邮箱，必填
  "password": "Password123",       // 密码，必填，≥8位
  "verifyCode": "123456",          // 验证码，必填，6位数字
  "role": "student",               // 角色，必填，枚举：student/teacher
  "name": "张三",                  // 姓名，必填，2-20字符
  "school": "XX大学"               // 学校，选填
}
```

**输出规范**:
```json
{
  "code": 0,
  "msg": "注册成功",
  "data": {
    "userId": 1001,
    "status": "active"             // 学生：active，教师：pending（待审核）
  }
}
```

**错误码**:
- `2001`: 账号已存在
- `2002`: 验证码错误
- `2003`: 密码强度不足
- `2004`: 参数格式错误

**验收标准**:
- [ ] 手机号格式验证（11位数字，1开头）
- [ ] 邮箱格式验证（符合RFC 5322标准）
- [ ] 密码强度验证（≥8位，含字母+数字）
- [ ] 验证码验证（6位数字，5分钟有效期）
- [ ] 教师注册后状态为pending，需管理员审核

#### 3.1.2 登录功能

**功能ID**: AUTH-002  
**功能名称**: 用户登录  
**优先级**: P0（必须）

**业务规则**:
1. 支持账号+密码登录
2. 支持验证码登录
3. 登录失败5次后锁定账号15分钟
4. 支持"记住密码"功能（7天有效期）
5. 登录成功返回JWT Token

**输入规范**:
```json
{
  "account": "13800138000",        // 手机号或邮箱，必填
  "password": "Password123",       // 密码，必填（密码登录时）
  "verifyCode": "123456",          // 验证码，必填（验证码登录时）
  "loginType": "password",         // 登录类型，枚举：password/verifyCode
  "rememberMe": true               // 记住密码，选填，默认false
}
```

**输出规范**:
```json
{
  "code": 0,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 604800,            // Token有效期（秒）
    "userInfo": {
      "id": 1001,
      "name": "张三",
      "role": "student",
      "avatar": "https://example.com/avatar.jpg",
      "school": "XX大学"
    }
  }
}
```

**错误码**:
- `1001`: 账号不存在
- `1002`: 密码错误
- `1003`: 账号已被锁定
- `1004`: 验证码错误
- `1005`: 账号待审核（教师）

**验收标准**:
- [ ] 登录失败次数限制（5次锁定15分钟）
- [ ] Token有效期：默认24小时，rememberMe=true时7天
- [ ] Token包含用户ID、角色、过期时间
- [ ] 登录成功后记录登录日志（IP、时间、设备）

### 3.2 学生端功能规范

#### 3.2.1 AI智能答疑

**功能ID**: STUDENT-001  
**功能名称**: AI智能答疑  
**优先级**: P0（必须）

**业务规则**:
1. 支持文本输入提问
2. 可选择学科分类（数学/计算机/英语/物理/化学等）
3. 支持多轮对话，保持上下文关联
4. 答案支持文字、公式（LaTeX）、步骤拆解
5. 支持对答案进行收藏、举报、评价
6. AI服务降级策略：主服务失败自动切换备用服务

**输入规范**:
```json
{
  "question": "如何求解一元二次方程？",  // 问题内容，必填，≤500字符
  "subject": "math",                    // 学科，选填，枚举
  "sessionId": "sess_123456",           // 会话ID，选填（多轮对话时必填）
  "context": [                          // 上下文，选填（多轮对话时自动填充）
    {
      "role": "user",
      "content": "前面的对话内容..."
    }
  ]
}
```

**输出规范**:
```json
{
  "code": 0,
  "msg": "成功",
  "data": {
    "answerId": "ans_789012",
    "answer": "一元二次方程的一般形式为：ax² + bx + c = 0...",
    "sessionId": "sess_123456",
    "steps": [                          // 解题步骤（可选）
      "步骤1：识别方程类型",
      "步骤2：应用求根公式..."
    ],
    "formulas": [                       // 公式（LaTeX格式）
      "x = \\frac{-b \\pm \\sqrt{b^2-4ac}}{2a}"
    ],
    "relatedQuestions": [               // 相关问题推荐
      "如何判断方程有实数解？"
    ],
    "aiProvider": "wenxin"              // 实际使用的AI服务商
  }
}
```

**学科枚举**:
```json
{
  "math": "数学",
  "computer": "计算机",
  "english": "英语",
  "physics": "物理",
  "chemistry": "化学",
  "other": "其他"
}
```

**性能要求**:
- 响应时间：≤3秒（95%场景）
- 上下文长度：≤4096 tokens
- 并发支持：≥20个同时请求

**验收标准**:
- [ ] 支持单轮对话
- [ ] 支持多轮对话（上下文关联）
- [ ] 答案包含公式时正确渲染LaTeX
- [ ] 答案可收藏、举报、评价
- [ ] AI服务降级策略生效（主服务失败自动切换）
- [ ] 响应时间≤3秒

## 四、接口规范

### 4.1 RESTful API 设计规范

#### 4.1.1 URL规范

**基础URL**: `https://api.ailearning.com/v1`

**资源命名**:
- 使用名词复数形式：`/users`, `/homeworks`, `/questions`
- 使用小写字母和连字符：`/lesson-plans`
- 层级关系：`/users/{userId}/homeworks`

**示例**:
```
GET    /v1/users                    # 获取用户列表
POST   /v1/users                    # 创建用户
GET    /v1/users/{userId}           # 获取单个用户
PUT    /v1/users/{userId}           # 更新用户
DELETE /v1/users/{userId}           # 删除用户
GET    /v1/users/{userId}/homeworks # 获取用户的作业列表
```

#### 4.1.2 请求规范

**请求头**:
```
Content-Type: application/json
Authorization: Bearer {token}        # 需要认证的接口
X-Request-ID: {uuid}                 # 请求追踪ID（可选）
```

**请求方法**:
- `GET`: 查询资源
- `POST`: 创建资源
- `PUT`: 更新资源（全量）
- `PATCH`: 更新资源（部分）
- `DELETE`: 删除资源

**分页参数**:
```json
{
  "page": 1,           // 页码，从1开始
  "pageSize": 20,      // 每页数量，默认20，最大100
  "sortBy": "createdAt",
  "sortOrder": "desc"  // 排序方向：asc/desc
}
```

**成功响应**:

## 七、安全规范

### 7.1 认证授权

**认证方式**: JWT (JSON Web Token)

**Token规范**:
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "userId": 1001,
    "role": "student",
    "exp": 1712822400,    // 过期时间
    "iat": 1712736000     // 签发时间
  }
}
```

**权限控制**: RBAC（基于角色的访问控制）

### 7.2 数据安全

| 安全措施 | 说明 |
|---------|------|
| 密码加密 | BCrypt算法，加盐哈希 |
| 敏感数据脱敏 | 手机号、邮箱中间位脱敏 |
| 数据传输加密 | HTTPS (TLS 1.2+) |
| 数据库加密 | 敏感字段AES加密 |

### 7.3 接口安全

| 安全措施 | 说明 |
|---------|------|
| 频率限制 | IP+Token限制，1分钟≤20次 |
| 防SQL注入 | 参数化查询 |
| 防XSS攻击 | 输入过滤、输出转义 |
| 防CSRF攻击 | Token校验 |
| 参数验证 | 类型、长度、格式验证 |

```yaml
version: '3.8'

services:

      - REDIS_HOST=redis
```
