## 1. 引言
### 1.1 文档目的
本需求说明书旨在明确“AI 学伴 - 轻量化团队 AI 辅助学习平台”的全量需求，包括功能、接口、性能、用户体验等维度，作为项目开发、测试、验收、维护的核心依据，确保开发成果符合高校师生及管理员的实际使用需求，同时满足项目开发流程规范。

### 1.2 项目背景
随着 AI+ 教育市场规模持续增长（2024 年超 3000 亿元，年增 25%），ChatGPT、文心一言等大语言模型降低了 AI 工具使用门槛，但现有产品存在“大型平台适配 K12 场景、专业工具功能单一”的问题。高校师生面临“学生答疑不及时、笔记整理效率低；教师出题/教案编写耗时、学生数据分散”等痛点。本项目聚焦高校场景，开发轻量化、多角色协同的 AI 辅助学习平台，解决上述痛点。

### 1.3 文档范围
覆盖平台的学生端、教师端、管理员端全功能模块，包含功能需求、接口需求、性能指标、用户需求、非功能需求、数据需求等，不包含项目管理（如进度、预算）相关内容。

### 1.4 术语与缩写
| 术语/缩写 | 说明 |
|-----------|------|
| AI API | 大语言模型开放接口（OpenAI API/百度文心一言/阿里通义千问） |
| FastAPI | 后端开发框架，用于构建高性能 API |
| Vue 3 | 前端渐进式框架，搭配 TypeScript 实现类型安全 |
| JWT | JSON Web Token，用于用户身份认证 |
| RESTful | 前后端交互的 API 设计规范 |
| 响应式布局 | 前端页面适配不同设备（PC/平板）的布局方式 |
| QPS | 每秒查询率，衡量接口并发处理能力 |

## 2. 目标用户

### 2.1 核心用户1：高校学生（本科/研究生）
需完成作业、整理笔记、分析错题、获取答疑

### 2.2 核心用户2：高校教师
需出题、管理教案/试题、跟踪学生学习情况

### 2.3 运维用户：平台管理员
需管理用户、配置系统、监控数据

## 3. 功能需求

### 3.1 通用功能需求（全角色）

#### 3.1.1 用户认证
- **注册**：1. 支持手机号/邮箱注册；2. 验证码校验；3. 密码强度校验（8位以上，含字母+数字）；4. 角色选择（学生/教师），管理员仅后台创建
- **登录**：1. 支持账号（手机号/邮箱）+ 密码登录；2. 验证码登录；3. 记住密码（7天有效期）；4. 登录失败次数限制（5次后锁定15分钟）
- **权限控制**：1. 基于角色的权限管理（RBAC）；2. 不同角色仅可见对应功能模块；3. 接口层面鉴权，防止越权访问
- **个人中心**：1. 基本信息编辑（头像、昵称、联系方式）；2. 密码修改/找回；3. 查看登录日志；4. 退出登录

#### 3.1.2 系统通知
- **消息推送**：1. 作业批改完成、试题发布、系统公告等消息推送；2. 支持未读标记、已读/删除操作；3. 消息保留90天

### 3.2 学生端功能需求

#### 3.2.1 AI 答疑
- **智能问答**：1. 支持文本输入提问，可选择学科（数学/计算机/英语等）；2. 多轮对话交互（上下文关联）；3. 答案支持文字、公式、步骤拆解；4. 支持答案收藏/举报/评价（五星+文字反馈）

#### 3.2.2 AI 笔记总结
- **笔记上传**：1. 支持文本粘贴、Word/PDF/图片格式文件上传（单文件≤50MB）；2. 支持批量上传（最多5个文件）
- **智能总结**：1. AI 自动提取知识点、重点、难点；2. 支持自定义总结维度（精简版/详细版/考点版）；3. 总结结果可编辑、导出（Word/PDF）

#### 3.2.3 AI 错题分析
- **错题录入**：1. 支持图片上传（拍照/相册）、文字输入错题内容；2. 支持手动标注错误位置；3. 关联对应知识点/学科
- **错误分析**：1. AI 分析错误原因（概念不清/计算错误/审题失误等）；2. 提供正确解题思路、步骤；3. 推荐3-5道同类练习题
- **错题本管理**：1. 按学科/知识点分类错题；2. 支持错题删除/标记已掌握；3. 错题本导出（Excel/PDF）

#### 3.2.4 作业提交与批改
- **作业查看**：1. 查看教师发布的作业列表（按截止时间排序）；2. 作业详情（题目、要求、附件）；3. 未完成/已完成状态区分
- **作业提交**：1. 支持文本、文件（Word/PDF/图片）提交；2. 截止时间前可多次修改提交；3. 提交后显示“待批改”状态
- **批改结果查看**：1. 查看 AI 批改得分、错误点标注；2. 查看错误解析、改进建议；3. 支持向教师提问（针对作业问题）

### 3.3 教师端功能需求

#### 3.3.1 AI 出题
- **试题生成**：1. 选择知识点、题型（选择/填空/简答/编程题）、难度（易/中/难）；2. 设置题量、总分；3. AI 自动生成试题，支持预览、编辑；4. 生成试题质量评分（AI 评估），支持筛选
- **试题编辑**：1. 手动新增/修改试题内容、答案、解析；2. 支持批量导入试题（Excel模板）；3. 试题标签（知识点/难度/题型）管理

#### 3.3.2 教案管理
- **教案创建**：1. 支持空白创建、模板创建；2. AI 辅助生成教案（输入主题、知识点，自动生成教学目标/流程/习题）；3. 支持富文本编辑（图片、公式、表格）
- **教案管理**：1. 教案按课程/学期分类；2. 支持教案复制/修改/删除/分享（仅同校教师）；3. 教案模板库（自定义模板保存）

#### 3.3.3 试题管理
- **题库管理**：1. 按学科/知识点/难度分类存储试题；2. 支持试题搜索（关键词/标签）；3. 批量导出/删除试题
- **自动组卷**：1. 选择题库、题型占比、难度分布，自动生成试卷；2. 试卷支持预览、编辑、保存、发布给学生

#### 3.3.4 学生管理
- **学生列表**：1. 查看所带班级学生信息；2. 支持按姓名/学号搜索、筛选；3. 导出学生信息（Excel）
- **学习跟踪**：1. 查看学生作业完成率、错题率、答疑次数；2. 生成学生学习报告（周/月维度）；3. 支持对学生进行一对一学习指导（留言/消息）
- **作业批改**：1. 查看学生提交的作业列表；2. 可手动批改（覆盖 AI 批改结果）；3. 批量批改/评语模板；4. 统计作业完成情况、平均分

### 3.4 管理员端功能需求

#### 3.4.1 用户管理
- **审核注册**：1. 查看学生/教师注册申请；2. 审核通过/驳回（填写驳回原因）；3. 批量审核
- **角色管理**：1. 创建/编辑/删除用户；2. 分配角色（学生/教师/管理员）；3. 禁用/启用账号；4. 重置用户密码

#### 3.4.2 系统配置
- **AI 接口管理**：1. 配置多厂商 AI API Key（OpenAI/百度/阿里）；2. 设置默认 AI 服务商；3. 配置 API 调用频率限制（按用户/按天）
- **系统参数**：1. 设置文件上传大小限制；2. 配置消息保留时长；3. 设置系统公告；4. 调整 AI 总结/分析的参数（如总结长度）
- **日志监控**：1. 查看系统操作日志（用户行为、接口调用、错误日志）；2. 日志按时间/用户/操作类型筛选；3. 日志导出（Excel），保留180天

#### 3.4.3 数据统计
- **平台使用统计**：1. 日/周/月活跃用户数、功能使用频次（答疑/出题/总结）；2. 可视化图表展示（折线图/柱状图）；3. 数据导出
- **AI 调用统计**：1. 按厂商/功能模块统计 AI 调用量；2. 统计调用成功率、平均响应时间；3. 预警（接近 API 额度上限时提醒）
- **教学效果分析**：1. 学生作业平均分、错题率趋势；2. 教师出题/教案发布量；3. 多维度数据报表生成与导出

#### 3.4.4 数据管理
- **备份与恢复**：1. 手动/自动备份数据库（自动备份每日凌晨，保留30天）；2. 支持备份文件下载、恢复操作；3. 备份操作日志记录

## 4. 接口需求

### 4.1 内部接口需求

#### 4.1.1 前后端交互接口（RESTful 规范）

**请求规范**：
1. 请求方法：GET（查询）/POST（创建）/PUT（更新）/DELETE（删除）
2. 请求头：携带 Token（Authorization: Bearer {JWT}）
3. 请求参数：JSON 格式，必填项非空校验

**示例（用户登录）**：
```
POST /api/v1/user/login
请求体：{
  "account": "13800138000",
  "password": "EncryptedPwd",
  "verifyCode": "123456"
}
```

**响应规范**：
1. 响应状态码：200（成功）/400（参数错误）/401（未授权）/403（禁止访问）/500（服务器错误）
2. 响应体：统一格式 `{"code":0,"msg":"成功","data":{}}`
3. 错误码：自定义（如 1001=账号不存在，1002=密码错误）

**示例（登录成功响应）**：
```json
{
  "code": 0,
  "msg": "登录成功",
  "data": {
    "token": "xxx",
    "userInfo": {
      "id": 1,
      "role": "student",
      "name": "张三"
    }
  }
}
```

#### 4.1.2 后端模块间接口

**控制器层 ↔ 服务层**：
- **AI 服务接口**：1. 统一封装 AI 调用逻辑，提供通用调用方法；2. 支持切换不同 AI 服务商；3. 异常捕获与降级处理

**服务层 ↔ 数据层**：
- **数据库操作接口**：1. 统一的 CRUD 封装；2. 支持事务管理（如作业提交+状态更新）；3. 慢查询监控（执行时间>1s 记录日志）

### 4.2 外部接口需求

#### 4.2.1 AI API 接口

**OpenAI API**：
- 调用 Chat Completions 接口，支持 gpt-3.5-turbo 模型
- 上下文长度 ≤ 4096 tokens
- 超时时间 ≤ 10s
- 降级策略：调用失败时自动切换至百度文心一言

**百度文心一言**：
- 调用 ERNIE-Bot 接口
- 支持学科定制化提示词
- 超时时间 ≤ 8s
- 降级策略：调用失败时自动切换至阿里通义千问

**阿里通义千问**：
- 调用通义千问通用接口
- 支持长文本总结（≤2万字）
- 超时时间 ≤ 8s
- 降级策略：调用失败时提示用户“当前 AI 服务繁忙，请稍后重试”

#### 4.2.2 其他外部接口

**数据库接口**：
1. 支持 MySQL 增删改查，索引优化
2. 连接池配置（最大连接数 50，最小空闲数 10）
3. 慢查询日志开启

**文件存储接口**：
1. 支持本地文件存储/对象存储（可选）
2. 文件上传/下载/删除接口
3. 文件 MD5 校验，防止重复上传

## 5. 性能指标需求

### 5.1 响应时间
1. 普通接口（登录、列表查询）：≤ 500ms
2. AI 接口（答疑、总结、出题）：≤ 3s（95% 场景）
3. 文件上传（≤10MB）：≤ 5s
4. 页面首次加载：≤ 2s（缓存后≤1s）

### 5.2 并发处理
1. 支持同时在线用户数 ≥ 100
2. 接口 QPS ≥ 50
3. AI 接口并发调用 ≥ 20（无明显延迟）

### 5.3 稳定性
1. 系统可用性 ≥ 99.5%（每月故障时间 ≤ 3.6小时）
2. AI 接口调用成功率 ≥ 98%（失败自动重试1次）
3. 数据库无死锁、无数据丢失

### 5.4 资源占用
1. 服务器 CPU 使用率 ≤ 70%（峰值）
2. 内存占用 ≤ 80%（峰值）
3. 磁盘使用率 ≤ 85%
4. 带宽占用 ≤ 80%（峰值）

### 5.5 数据处理
1. 单批次笔记总结（5万字）：≤ 10s
2. 单批次错题分析（10道题）：≤ 8s
3. 自动组卷（50道题）：≤ 5s

## 6. 用户需求

### 6.1 学生用户需求
- **核心痛点解决**：1. 课后问题即时解答，无需等待教师回复；2. 笔记自动总结，快速提炼重点，节省整理时间；3. 错题精准分析，针对性提升薄弱知识点；4. 作业批改结果即时查看，快速了解错误原因
- **体验需求**：1. 操作简单，核心功能（答疑/总结/错题）操作步骤 ≤ 3步；2. 界面简洁，无冗余信息，重点功能突出；3. 答案/总结结果可编辑、导出，满足个性化需求；4. 支持多设备访问（PC/平板）
- **成本需求**：1. 平台免费使用，仅消耗 AI API 基础额度（高校可补贴）；2. 无强制付费功能；3. 低网络带宽即可使用

### 6.2 教师用户需求
- **核心痛点解决**：1. 减少重复性答疑工作，将精力投入教学准备；2. 快速生成高质量试题/教案，提升工作效率；3. 批量批改作业，节省批改时间；4. 跟踪学生学习数据，实现个性化教学
- **体验需求**：1. 出题/教案生成支持自定义，满足不同课程需求；2. 学生数据可视化，直观了解班级学习情况；3. 支持批量操作（导入试题/批改作业），提升效率；4. 教案/试题可复用、可分享，方便协作
- **管理需求**：1. 学生信息批量管理，支持按班级/学号筛选；2. 作业发布/批改状态清晰，便于跟踪；3. 学习数据可导出，用于教学分析

### 6.3 管理员用户需求
- **运维需求**：1. 便捷管理用户账号，审核/禁用/启用操作简单；2. 系统配置可视化，无需修改代码；3. 日志监控全面，便于定位问题；4. 数据备份/恢复操作安全、便捷
- **统计需求**：1. 多维度数据统计，支持自定义时间范围；2. 数据可视化展示，便于汇报；3. AI 调用量监控，防止费用超支；4. 平台使用情况分析，指导功能优化
- **安全需求**：1. 系统操作日志可追溯，满足审计要求；2. 敏感操作（如权限修改）需二次确认；3. 数据加密存储，防止泄露

## 7. 非功能需求

### 7.1 安全性需求

#### 7.1.1 数据安全
1. 用户密码加密存储（BCrypt 算法）
2. 敏感数据（手机号/邮箱）脱敏展示
3. 数据库传输加密（SSL）
4. 文件上传病毒扫描（防止恶意文件）

#### 7.1.2 接口安全
1. 接口防刷（IP+Token 频率限制，1分钟≤20次）
2. 防 SQL 注入（参数化查询）
3. 防 XSS 攻击（前端输入过滤、后端转义）
4. 防 CSRF 攻击（Token 校验）

#### 7.1.3 权限安全
1. 最小权限原则，不同角色仅拥有必要权限
2. 管理员操作日志全程记录
3. 重要操作（如数据删除）需二次确认

### 7.2 易用性需求

#### 7.2.1 操作便捷性
1. 核心功能操作步骤 ≤ 3步
2. 界面引导清晰（新手引导、功能提示）
3. 常用功能支持快捷键（如 Ctrl+S 保存）
4. 错误提示明确，提供解决方案

#### 7.2.2 界面设计
1. 视觉风格统一（颜色、字体、按钮样式）
2. 符合高校师生审美，简洁专业
3. 重要信息突出（如作业截止时间、未读消息）
4. 支持暗黑模式（可选）

#### 7.2.3 帮助支持
1. 内置帮助中心（常见问题、操作指南）
2. 功能模块提供“使用说明”弹窗
3. 支持在线反馈（问题提交、建议收集）

### 7.3 可维护性需求

#### 7.3.1 代码规范
1. 前端遵循 Vue 3 官方风格指南，后端遵循 FastAPI 规范
2. 代码注释率 ≥ 30%（核心模块）
3. 变量/函数命名语义化，避免硬编码

#### 7.3.2 模块化设计
1. 前端按功能模块拆分组件，后端按业务分层（控制器/服务/模型）
2. 通用逻辑封装为工具函数/公共组件
3. 配置与代码分离（环境变量管理）

#### 7.3.3 日志规范
1. 分级日志（DEBUG/INFO/WARN/ERROR）
2. 错误日志包含堆栈信息、请求参数
3. 日志按天分割，便于检索

### 7.4 兼容性需求

#### 7.4.1 浏览器兼容
支持 Chrome ≥ 90、Firefox ≥ 88、Edge ≥ 90，功能无缺失、界面无错位

#### 7.4.2 设备兼容
支持 PC（Windows/macOS/Linux）、平板（iPad/安卓平板），响应式布局适配

#### 7.4.3 数据兼容
1. 支持主流文件格式（Word/PDF/Excel/PNG/JPG）
2. 导出文件兼容 Office 2016+ 版本

### 7.5 可扩展性需求

#### 7.5.1 功能扩展
1. 预留接口（如第三方登录、支付接口）
2. 模块解耦，新增功能不影响现有逻辑
3. 支持插件化开发（如新增学科模板）

#### 7.5.2 AI 扩展
1. 支持新增 AI 服务商接口（如科大讯飞）
2. 提示词模板可配置，便于适配不同学科
3. AI 功能参数可调整（如总结长度、出题难度）

#### 7.5.3 部署扩展
1. 支持单机/集群部署
2. 支持容器化部署（Docker）
3. 支持云服务器/本地服务器部署
